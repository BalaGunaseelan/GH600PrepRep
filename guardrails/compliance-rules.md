# Compliance Guardrails

## Why this file exists
This file demonstrates that compliance can also be expressed as repository policy. GH-600 learners should understand when agents must defer to humans for regulated decisions.

## Allowed actions
- Document review checkpoints and approval expectations.
- Use fictional examples that avoid personal or customer data.
- Version-control educational policies alongside code.

## Denied actions
- Claim legal or regulatory compliance without evidence.
- Store personal, medical, or financial data in examples.
- Bypass required human approvals for sensitive changes.

## Approval-required actions
- Publishing policy to external stakeholders
- Mapping repository controls to a real regulatory framework
- Accepting residual risk after a failed control

## Real-project usage
A real program would attach these rules to specific regulations, control owners, and evidence collection processes.
