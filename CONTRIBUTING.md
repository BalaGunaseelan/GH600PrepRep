# Contributing Guide

## Why this file exists
This file demonstrates how contributors can be guided toward the learning goal of the repository. GH-600 learners should see that contribution policy is another important GitHub artifact.

## How to contribute
1. Keep changes educational and easy to inspect.
2. Prefer the smallest possible diff that satisfies the request.
3. Avoid adding real credentials, private endpoints, or production-only complexity.
4. Run `python -m unittest discover -s tests -v` when code changes are made.

## Pull request expectations
- Explain which GH-600 concept the change demonstrates.
- Update README when you add a new top-level educational folder.
- Note any human approval or escalation requirement triggered by the change.
