# Long-Term Memory Demonstration

## Why this file exists
This file demonstrates durable memory that can outlive a single task. GH-600 learners should see how repository conventions become reusable context for future agent runs.

## What memory stores
- Stable repository conventions
- Approved workflow patterns
- Preferred validation commands
- Common escalation contacts or roles

## Lifecycle
Long-term memory persists across sessions until it is updated or deleted by a human-maintained process.

## Examples
- "Use Python standard-library unittest for this demo repository."
- "Never place real tokens in MCP examples."
- "README must explain each educational folder."

## Real-project usage
In real systems, durable memory improves consistency but must be governed so stale or risky facts can be corrected.
