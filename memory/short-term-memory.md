# Short-Term Memory Demonstration

## Why this file exists
This file explains transient agent memory in a way learners can inspect directly in the repository. It helps GH-600 candidates connect memory concepts to tangible artifacts.

## What memory stores
- Current task goal
- Recently opened files
- Temporary validation notes
- Open questions waiting for clarification

## Lifecycle
Short-term memory exists for the active session or task window. It is expected to expire or be discarded once the agent finishes the work.

## Examples
- "The current issue asks for a hello-world Python sample."
- "The last test run passed with two assertions."
- "The reviewer asked for more comments in workflow files."

## Real-project usage
Real short-term memory helps agents stay coherent during multi-step work without polluting long-lived knowledge stores.
