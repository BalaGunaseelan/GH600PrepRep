# Security Policy

## Why this file exists
This file demonstrates where repository-level security guidance usually lives. It supports GH-600 topics around safe automation, disclosure handling, and guardrails.

## Reporting a vulnerability
Because this repository is educational, do not include real secrets or exploit details in public issues. Instead, open a private security report through normal GitHub security reporting channels for the owning account.

## Supported content
The repository intentionally contains placeholder configurations and non-production examples only.

## Real-project usage
A production `SECURITY.md` would define supported versions, disclosure timelines, and response expectations.
